1. Add `getLawyers` method to `src/data/repositories/UserRepository.js`:
   ```javascript
   /**
    * Get all users with lawyer role for the directory
    * @param {Object} options - Filter options
    * @param {number} options.limitCount - Max results (default 50)
    * @returns {Promise<Object[]>} Array of lawyer user objects
    */
   async getLawyers({ limitCount = 50 } = {}) {
     return this.firestoreDataSource.query(COLLECTIONS.USERS, {
       where: [['role', '==', 'lawyer']],
       orderBy: [['displayName', 'asc']],
       limit: limitCount,
     });
   }
   ```
   Note: Firestore cannot do array-contains + array-contains in the same query, so specialization and language filters will be applied client-side. The user population of lawyers is small enough that fetching all lawyers and filtering client-side is appropriate.

2. Add `getLawyerReviews` method to UserRepository:
   ```javascript
   /**
    * Get reviews for a lawyer from their reviews subcollection
    * @param {string} lawyerId
    * @returns {Promise<Object[]>}
    */
   async getLawyerReviews(lawyerId) {
     return this.firestoreDataSource.query(
       `${COLLECTIONS.USERS}/${lawyerId}/${SUBCOLLECTIONS.REVIEWS}`,
       { orderBy: [['createdAt', 'desc']], limit: 20 }
     );
   }
   ```

3. Create `src/presentation/hooks/legal/useLawyerDirectory.js`:
   - Import container, useState, useEffect, useCallback
   - State: `lawyers` (array), `loading` (bool), `error` (null|string), `filters` object with `search` (string), `specialization` (string|null), `availability` (bool|null), `language` (string|null)
   - On mount: call `container.getUserRepository().getLawyers()` to fetch all lawyers
   - Apply client-side filtering:
     - `search`: filter by displayName or companyName (case-insensitive includes)
     - `specialization`: filter where lawyer's `specializations` array includes the value
     - `availability`: filter where `isAvailable === true`
     - `language`: filter where lawyer's `languages` array includes the value
   - Sort results by: availability first (available lawyers at top), then by rating (if present) descending, then alphabetically
   - Expose: `{ lawyers, loading, error, filters, setFilter, clearFilters }`
   - `setFilter(key, value)` updates the filters object
   - `clearFilters()` resets all filters to defaults
   - The filtering is memoized (useMemo) to avoid re-filtering on every render

   NOTE: Lawyer user documents need these fields to support the directory:
   - `specializations`: string[] (e.g., ['trade-law', 'maritime-law', 'contract-law'])
   - `languages`: string[] (e.g., ['English', 'Turkish'])
   - `yearsOfExperience`: number
   - `education`: string
   - `isAvailable`: boolean
   - `rating`: number (average, computed from reviews)
   - `reviewCount`: number
   These are set during onboarding or admin setup. The directory code reads them but does not create them.
