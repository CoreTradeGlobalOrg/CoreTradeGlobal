   - Follows `sendDealNotifications` pattern exactly
   - Creates in-app notification at `users/{recipientId}/notifications` with:
     - type: 'legal'
     - eventType (string: 'hire_request', 'hire_accepted', 'hire_declined', 'engagement_completed', 'new_message', 'new_draft', 'risk_update')
     - title: derived from eventType via a `getLegalEventTitle(eventType)` helper
     - body: derived from eventType + dealProductName via `getLegalEventBody(eventType, dealProductName)` helper
     - engagementId, dealId
     - isRead: false, createdAt: Timestamp.now()
     - link: `/deals/${dealId}/legal`
   - Sends email via `sendDealEmail()` (non-blocking, same pattern: try/catch, log error, continue)
   - Email subject/body formatted for legal events (simple HTML with CTG branding matching existing deal email templates)

3. Create `exports.hireLayyer = onCall(async (request) => { ... })`:
   - Validates: request.auth exists, destructures { dealId, lawyerId } from request.data
   - Reads deal doc to verify caller is buyerId or sellerId (deal participant)
   - Reads lawyer user doc to verify role === 'lawyer'
   - Uses deterministic engagement ID: `${dealId}_${request.auth.uid}` -- prevents duplicate engagements per client per deal (Research Pitfall 5)
   - Checks if engagement doc already exists -- if so and status is 'active' or 'pending', throw HttpsError('already-exists', 'You already have an active legal engagement for this deal')
   - If exists and completed, allow re-hire (delete old doc or overwrite)
   - Creates engagement doc at `legalEngagements/${dealId}_${request.auth.uid}`:
     ```javascript
     {
       clientId: request.auth.uid,
       lawyerId,
       dealId,
       participants: [request.auth.uid, lawyerId],
       dealProductName: deal.productName || deal.product?.name || 'Deal',
       clientDisplayName: callerUser.displayName || callerUser.companyName || 'Client',
       lawyerDisplayName: lawyerUser.displayName || lawyerUser.companyName || 'Lawyer',
       status: 'pending',
       createdAt: Timestamp.now(),
       updatedAt: Timestamp.now(),
     }
     ```
   - Updates deal doc: `arrayUnion` to `lawyerIds` field -- enables lawyer to read deal via extended rule from Plan 01
   - Calls `sendLegalNotification` to lawyer: eventType 'hire_request'
   - Returns { engagementId, status: 'pending' }

4. Create `exports.respondToHireRequest = onCall(async (request) => { ... })`:
   - Validates: request.auth exists, destructures { engagementId, action } from request.data (action: 'accept' | 'decline')
   - Uses runTransaction:
     - Reads engagement doc, verifies caller is lawyerId
     - Verifies status === 'pending'
     - If accept: update status to 'active', updatedAt
     - If decline: update status to 'declined', updatedAt (NOTE: 'declined' is a terminal sub-state of pending; the client can then re-hire a different lawyer)
   - If accepted: add system message to legalMessages subcollection: "Legal consulting session started. All conversations are encrypted and recorded."
   - Sends notification to client: eventType 'hire_accepted' or 'hire_declined'
   - Returns { status: newStatus }

5. Create `exports.closeLegalEngagement = onCall(async (request) => { ... })`:
   - Validates: request.auth exists, destructures { engagementId } from request.data
   - Uses runTransaction:
     - Reads engagement doc, verifies caller is in participants array
     - Verifies status === 'active'
     - Updates status to 'completed', updatedAt
   - Adds system message: "Legal consulting session ended. This channel is now read-only."
   - Sends notification to the OTHER participant (not the caller): eventType 'engagement_completed'
   - Returns { status: 'completed' }

6. Create `exports.submitLawyerReview = onCall(async (request) => { ... })`:
   - Validates: request.auth exists, destructures { engagementId, rating, comment } from request.data
   - Reads engagement doc, verifies caller is clientId (only clients can review)
   - Verifies status === 'completed' (can only review after engagement ends)
   - Validates rating is 1-5 integer, comment is string (optional, max 1000 chars)
   - Writes review to `users/{lawyerId}/reviews/{auto-id}`:
     ```javascript
     {
       reviewerId: request.auth.uid,
       reviewerName: clientUser.displayName || 'Anonymous',
       engagementId,
       dealId: engagement.dealId,
       rating,
       comment: comment || '',
       createdAt: Timestamp.now(),
     }
     ```
   - Returns { success: true }

All notification side effects MUST be called OUTSIDE any transactions (project pattern: prevents duplicate sends on retry).
