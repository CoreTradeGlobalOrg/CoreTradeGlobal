**Server-side transaction (functions/index.js):**

In the hireLayyer Cloud Function (lines 3619-3648), wrap the existence check + write in a Firestore transaction. Currently the code does a standalone `get()` then a standalone `set()` — two concurrent requests can both pass the existence check and both write.

Replace the existing code block (lines 3618-3648) with a transaction:
```javascript
const result = await db.runTransaction(async (t) => {
  const existingDoc = await t.get(engagementRef);
  if (existingDoc.exists) {
    const existing = existingDoc.data();
    if (existing.status === ENGAGEMENT_STATUS.PENDING || existing.status === ENGAGEMENT_STATUS.ACTIVE) {
      throw new HttpsError(
        'already-exists',
        'You already have an active legal engagement for this deal.'
      );
    }
    // completed or declined: allow re-hire (overwrite)
  }

  t.set(engagementRef, {
    clientId: uid,
    lawyerId,
    dealId,
    participants: [uid, lawyerId],
    dealProductName,
    clientDisplayName,
    lawyerDisplayName,
    status: ENGAGEMENT_STATUS.PENDING,
    createdAt: now,
    updatedAt: now,
  });

  return { engagementId, status: ENGAGEMENT_STATUS.PENDING };
});
```

Keep the dealRef.update (lawyerIds arrayUnion) and sendLegalNotification call OUTSIDE the transaction (they are side effects that should not cause transaction retries). The `return result;` replaces the current `return { engagementId, status: ENGAGEMENT_STATUS.PENDING };` at line 3660.
