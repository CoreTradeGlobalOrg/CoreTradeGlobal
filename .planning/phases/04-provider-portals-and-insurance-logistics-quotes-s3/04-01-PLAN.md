Create 5 new files following established patterns from contractConstants.js, Contract.js, and ContractRepository.js:

**1. `src/core/constants/quoteConstants.js`** — Define all enums and options:
- `QUOTE_REQUEST_STATUS`: pending, quoted, declined, selected, not_selected
- `QUOTE_STATUS`: active, withdrawn, expired, accepted
- `ICC_COVERAGE`: A/B/C with value, label, description (A = All Risks, B = Named Perils Extended, C = Named Perils Basic)
- `COVERAGE_SCOPE`: warehouse_to_warehouse, port_to_port, door_to_door (value + label)
- `TRANSPORT_MODE`: sea, air, road, rail, multimodal (value + label + lucide icon name: Ship, Plane, Truck, Train, ArrowLeftRight)
- `CONTAINER_TYPE`: array of { value, label } for 20ft, 40ft, 40ft_hc, 45ft_hc, lcl, reefer, open_top, flat_rack
- `CAPABILITY_TAGS`: string array ['GPS Tracking', 'Cold Chain', 'Door-to-Door', 'Customs Support', 'Hazmat Certified', 'Express Delivery', 'Insurance Included', 'Bonded Warehouse']
- `QUOTE_VALIDITY_OPTIONS`: array of { value (hours), label } for 12h, 24h, 48h, 72h
- `QUOTE_REQUEST_DEADLINE_HOURS`: 72 (constant for provider response deadline)
- Export all individually and as default object

**2. `src/domain/entities/QuoteRequest.js`** — QuoteRequest class:
- Constructor takes: id, dealId, providerUid, providerType ('insurance'|'logistics'), dealSnapshot (object), buyerId, sellerId, status, deadline, createdAt, updatedAt
- `static fromFirestore(data)` — converts Timestamps (deadline, createdAt, updatedAt) to Date via `.toDate?.()` pattern
- `isPending()`, `isQuoted()`, `isDeclined()`, `isSelected()` — status helpers
- `isExpiredDeadline()` — checks if deadline has passed (Date.now() > deadline)
- `getKanbanColumn()` — returns 'newRequests' for pending, 'quoted' for quoted, 'declined' for declined, 'selected' for selected|not_selected
- `isInsurance()`, `isLogistics()` — providerType checks

**3. `src/domain/entities/Quote.js`** — Quote class:
- Constructor takes all insurance + logistics fields (see RESEARCH.md Quote entity code example): id, requestId, dealId, providerUid, providerType, iccCoverage, warClause, strikesClause, premiumAmount, coverageAmount, deductiblePct, claimsPaymentDays, policyStartDate, policyEndDate, coverageScope, certificateType, transportMode, containerType, freightCost, estimatedTransitDays, loadingDate, estimatedArrival, capabilityTags, currency, notes, validUntil, status, createdAt, updatedAt
- Null defaults for type-specific fields (insurance fields null for logistics and vice versa)
- `static fromFirestore(data)` — converts all Timestamp fields via `.toDate?.()` pattern
- `isExpired()` — checks validUntil against Date.now()
- `isActive()` — status === 'active' AND !isExpired()
- `isInsurance()`, `isLogistics()` — providerType checks
- `getPrice()` — returns premiumAmount for insurance, freightCost for logistics

**4. `src/data/repositories/QuoteRequestRepository.js`** — following ContractRepository pattern:
- Constructor takes firestoreDataSource (same DI pattern)
- `subscribeToRequestsForProvider(providerUid, callback)` — query quoteRequests where providerUid == uid, orderBy createdAt desc, onSnapshot maps to QuoteRequest.fromFirestore
- `subscribeToRequestsForDeal(dealId, callback)` — query quoteRequests where dealId == id, orderBy createdAt desc, onSnapshot maps to QuoteRequest.fromFirestore
- Both return the unsubscribe function from onSnapshot
- Import collection, query, where, orderBy, onSnapshot from firebase/firestore
- Import db from @/core/config/firebase.config
- Import QuoteRequest from @/domain/entities/QuoteRequest

**5. `src/data/repositories/QuoteRepository.js`** — following same pattern:
- Constructor takes firestoreDataSource
- `subscribeToQuotesForRequest(requestId, callback)` — query quoteRequests/{requestId}/providerQuotes subcollection, orderBy createdAt desc, onSnapshot maps to Quote.fromFirestore
- `subscribeToQuotesForDeal(dealId, callback)` — use collectionGroup('providerQuotes') with where dealId == id, orderBy createdAt desc (NOTE: requires composite index). Maps to Quote.fromFirestore
- IMPORTANT: Use subcollection name `providerQuotes` (not `quotes`) to avoid collision with existing `requests/{id}/quotes` subcollection per Research Pitfall 7
- Both return unsubscribe function
