- Receives (dealId, dealData) as parameters
- Queries `users` collection for all insurance_provider and logistics_provider role users
- Creates one quoteRequest document per provider in `quoteRequests` collection using db.batch()
- Insurance dealSnapshot includes ALL deal fields from latestOfferSnapshot: productName, productImage, quantity, unit, incoterm, namedPlace, paymentTerms, deliveryDeadline, currency, price, estimatedTotal
- Logistics dealSnapshot: build using EXPLICIT ALLOWLIST (not spread-then-delete). Only include: productName, productImage, quantity, unit, incoterm, namedPlace, paymentTerms, deliveryDeadline, currency. NEVER include price or estimatedTotal.
- Each quoteRequest document shape: { dealId, providerUid, providerType, dealSnapshot, buyerId, sellerId, status: 'pending', deadline: now + 72h, createdAt, updatedAt }
- Log count of created requests
- IMPORTANT: This function runs OUTSIDE any transaction (same pattern as sendDealNotifications) — do NOT put inside a transaction

**Step 3: Hook broadcastQuoteRequests into existing onDealStatusChanged:**

**Step 4: Add `submitQuote` Cloud Function (onCall):**
- Receives { requestId, quoteData } from request.data
- Auth check: uid must exist
- Read the quoteRequest document; verify provider owns it (quoteRequest.providerUid === uid)
- Verify request status is 'pending' or 'quoted' (allow edits)
- Verify request deadline has not passed
- Validate quoteData based on providerType:
  - Insurance: require iccCoverage (A/B/C), premiumAmount (> 0), coverageAmount (> 0), deductiblePct (0-100), claimsPaymentDays (> 0), policyStartDate, policyEndDate, coverageScope, validityHours (12/24/48/72)
  - Logistics: require transportMode (sea/air/road/rail/multimodal), freightCost (> 0), estimatedTransitDays (> 0), loadingDate, estimatedArrival, validityHours
  - Optional for both: notes, currency (default 'USD')
  - Insurance optional: warClause, strikesClause, certificateType
  - Logistics optional: containerType, capabilityTags
- Compute validUntil as Timestamp.fromMillis(Date.now() + validityHours * 60 * 60 * 1000)
- Check if a quote already exists for this request (query providerQuotes subcollection where providerUid == uid)
  - If exists: UPDATE the existing quote document (edit scenario)
  - If not: CREATE new quote in quoteRequests/{requestId}/providerQuotes subcollection
- Quote document includes: requestId, dealId, providerUid, providerType, all validated fields, validUntil, status: 'active', createdAt, updatedAt
- Denormalize buyerId and sellerId from quoteRequest onto quote doc (for Firestore rules to avoid get() per Pitfall 4)
- Update quoteRequest status to 'quoted' and updatedAt
- Send notification to buyer: use existing sendDealNotifications pattern adapted for quote events. Event type: 'quote_received'. If notification adaptation is complex, use a simple in-app notification write to buyer's notifications subcollection instead.
- Return { success: true }

**Step 5: Add `acceptQuote` Cloud Function (onCall):**
- Receives { quoteRequestId, quoteId } from request.data
- Auth check: uid must exist
- Use runTransaction (CRITICAL for atomicity):
  - Read quoteRequest and quote documents inside transaction
  - Authorization: uid must equal quoteRequest.buyerId
  - SERVER-SIDE EXPIRY CHECK (QUOTE-04): `quote.validUntil.toMillis() <= Timestamp.now().toMillis()` → throw HttpsError('failed-precondition', 'This quote has expired')
  - Status guard: quote.status must be 'active'
  - Update quote status to 'accepted'
  - Update quoteRequest status to 'selected'
  - Update deal document: set `selectedInsuranceQuoteId` or `selectedLogisticsQuoteId` (based on providerType) and corresponding `RequestId` field
- After transaction: notify winning provider ('quote_accepted') and buyer confirmation
- Return { success: true }

**Step 5: Add Firestore rules for quoteRequests and providerQuotes to firestore.rules:**

```
match /quoteRequests/{requestId} {
  // Assigned provider, buyer, seller, or admin can read
  allow read: if isAuthenticated() && (
    isAdmin() ||
    request.auth.uid == resource.data.providerUid ||
    request.auth.uid == resource.data.buyerId ||
    request.auth.uid == resource.data.sellerId
  );
  // All writes via Cloud Functions (Admin SDK bypasses rules)
  allow write: if false;

  // providerQuotes subcollection
  match /providerQuotes/{quoteId} {
    // Provider who owns the quote, buyer, seller, or admin can read
    // Uses denormalized buyerId/sellerId on quote doc (avoids get() per Pitfall 4)
    allow read: if isAuthenticated() && (
      isAdmin() ||
      request.auth.uid == resource.data.providerUid ||
      request.auth.uid == resource.data.buyerId ||
      request.auth.uid == resource.data.sellerId
    );
    allow write: if false;
  }
}
```
