<objective>
Build the provider portal UI: a kanban-style dashboard at /provider/dashboard where insurance and logistics providers view incoming quote requests grouped by status (New, Quoted, Declined, Selected), and submit quotes via type-specific forms. Includes hooks for real-time subscriptions and Cloud Function action wrappers.

Purpose: Providers need a dedicated workspace to manage their quote requests efficiently. The kanban layout gives immediate visual status overview. Per user decision: shared layout for both provider types, only quote form fields differ.

Output: 10 files — 7 new (route, 4 components, 2 hooks), 2 modified (provider redirect, navbar link), plus the QuoteDetailView component for the quote form side-by-side layout
</objective>

    declined: requests.filter(r => r.getKanbanColumn() === 'declined'),
