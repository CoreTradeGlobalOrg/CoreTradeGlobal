  <name>Task 1: Fix Firestore offers subcollection security rule for member access</name>
  <files>firestore.rules</files>
  <action>
The root cause: in `firestore.rules` lines 254-258, the offers subcollection rule calls `isDealParticipant()` (defined at line 241-243). This function reads `resource.data.buyerId` and `resource.data.sellerId`. But inside the offers subcollection match (`match /offers/{offerId}`), `resource.data` refers to the **offer document**, which does NOT have `buyerId` or `sellerId` fields. These fields exist only on the parent deal document. This causes all non-admin member reads to fail with "Missing or insufficient permissions."

Fix the offers subcollection rule at line 256 to use `get()` to read the parent deal document instead of relying on `resource.data`:

Replace:
```
match /offers/{offerId} {
  allow read: if isAuthenticated() && isDealParticipant();
  allow write: if false;
}
```

With:
```
match /offers/{offerId} {
  allow read: if isAuthenticated() && (
    request.auth.uid == get(/databases/$(database)/documents/deals/$(dealId)).data.buyerId ||
    request.auth.uid == get(/databases/$(database)/documents/deals/$(dealId)).data.sellerId
  );
  allow write: if false; // All writes via Cloud Functions
}
```

Note: This uses one `get()` call per rule evaluation. Firestore allows up to 10 `get()` calls per rule chain. Since this is the only `get()` in the deals path, this is well within limits. The global admin override (`match /{document=**} { allow read, write: if isAdmin(); }`) already handles admin access, so no `isAdmin()` check is needed here.

Do NOT change the `isDealParticipant()` function itself or the parent deal document rule -- those work correctly because `resource.data` IS the deal document in that context.

**Bug 1 - dealConstants.js: Add UNECE-to-DEAL_UNIT mapping**

Products store units as UNECE codes (KGM, TNE, PCE, MTR, MTK, CH, etc. from `src/core/constants/units.js`). The deal form uses DEAL_UNITS which have different values (kg, ton, pieces, metre, m2, containers). Add a mapping object after the DEAL_UNITS array:

```js
/**
 * Maps UNECE product unit codes to DEAL_UNITS values.
 * Used when pre-filling the deal form from product data.
 * Products use UNECE codes (KGM, TNE, PCE); deals use simplified values (kg, ton, pieces).
 */
export const UNECE_TO_DEAL_UNIT = {
  KGM: 'kg',
  GRM: 'kg',      // Gram -> closest deal unit is kg
  TNE: 'ton',
  LBR: 'kg',      // Pound -> closest deal unit is kg
  PCE: 'pieces',
  SET: 'pieces',   // Set -> closest is pieces
  PR: 'pieces',    // Pair -> closest is pieces
  DZN: 'pieces',   // Dozen -> closest is pieces
  GRO: 'pieces',   // Gross -> closest is pieces
  KIT: 'pieces',   // Kit -> closest is pieces
  MTR: 'metre',
  CMT: 'metre',    // Centimeter -> closest is metre
  MMT: 'metre',    // Millimeter -> closest is metre
  FOT: 'metre',    // Foot -> closest is metre
  INH: 'metre',    // Inch -> closest is metre
  MTK: 'm2',
  CMK: 'm2',       // Square centimeter -> closest is m2
  CH: 'containers',
  CT: 'containers', // Carton -> closest is containers
  PX: 'containers', // Pallet -> closest is containers
  PK: 'containers', // Package -> closest is containers
};
```

Also add it to the default export at the bottom of the file.

**Bug 2 - deals/new/page.jsx: Read product.stockQuantity and map unit**

In `src/app/(main)/deals/new/page.jsx`, at the `defaultValues` object (around line 159-164), change:

From:
```js
const defaultValues = {
  price: product?.price && product.price > 0 ? product.price : undefined,
  currency: product?.currency || 'USD',
  quantity: undefined,
  unit: product?.unit || '',
};
```

To:
```js
import { UNECE_TO_DEAL_UNIT } from '@/core/constants/dealConstants';

// ... (add import at top of file with other imports)

const defaultValues = {
  price: product?.price && product.price > 0 ? product.price : undefined,
  currency: product?.currency || 'USD',
```

This reads `product.stockQuantity` for the quantity default and maps the UNECE unit code (e.g. `KGM`) to the DEAL_UNITS value (e.g. `kg`) via the mapping. If no mapping exists, it falls back to the raw product unit.
