    **1. Add Cloud Functions to `functions/index.js`:**

    All state-changing operations go through `onCall` Cloud Functions using `db.runTransaction()` (Admin SDK = pessimistic locking). Follow existing CF patterns in the file (v2 imports, request.auth pattern, HttpsError).

    **`createDeal` (onCall):**
    - Auth check: must be logged in
    - Accepts: `{ conversationId, productId, initialOffer }` where initialOffer contains all offer fields
    - Fetches product doc to get seller ID + denormalized product data (name, image, category)
    - Determines buyer/seller: product owner = seller always. If initiator is the product owner, the other party in the conversation is the buyer (fetch conversation doc to find the other participant)
    - Uses `db.runTransaction`:
      - Creates `deals/{autoId}` with status 'negotiating', round 1, currentTurnUid = other party
      - Creates `deals/{dealId}/offers/{autoId}` with round 1, status 'open', all offer terms, estimatedTotal = price * quantity, expiresAt = now + expiryHours
      - Sets `latestOfferSnapshot` on deal doc (denormalized for list page performance)
    - Post-transaction: sends system message to conversation ("Deal initiated: [productName]" with link `/deals/{dealId}`)
    - Returns `{ success: true, dealId }`

    **`submitCounterOffer` (onCall):**
    - Auth check + accepts `{ dealId, offer, expectedRound }`
    - `db.runTransaction`:
      - Read deal + latest offer (offer with status 'open')
      - Guard: `deal.currentTurnUid === uid` (must be receiver's turn)
      - Guard: `deal.status === 'negotiating'`
      - Guard: latest offer status === 'open'
      - Guard: `deal.round === expectedRound` (prevents stale writes)
      - Update old offer status to 'countered'
      - Create new offer with round + 1, status 'open', new expiresAt
      - Update deal: round + 1, currentTurnUid flipped to other party, latestOfferSnapshot updated
    - Returns `{ success: true, dealId }`

    **`acceptOffer` (onCall):**
    - Auth + accepts `{ dealId, offerId }`
    - `db.runTransaction`:
      - Read deal + offer
      - Guard: `deal.currentTurnUid === uid`
      - Guard: `offer.status === 'open'`
      - Guard: `deal.status === 'negotiating'`
      - Guard: offer not expired (`expiresAt > now`)
      - Update offer status to 'accepted', deal status to 'accepted'
    - Returns `{ success: true }`
    - Note: Phase 3 will trigger contract generation via `onDocumentUpdated` on deal status change — this CF only sets the status

    **`rejectOffer` (onCall):**
    - Same structure as acceptOffer
    - Guards: same (currentTurnUid, offer open, deal negotiating)
    - Updates offer status to 'rejected', deal status to 'rejected'

    **`withdrawOffer` (onCall):**
    - Auth + accepts `{ dealId, offerId }`
    - Guard: `offer.submittedBy === uid` (only sender can withdraw)
    - Guard: `offer.status === 'open'` (can only withdraw open offers)
    - Guard: `deal.status === 'negotiating'`
    - Updates offer status to 'withdrawn', deal status to 'withdrawn'

    All CFs: DO NOT call Resend or send notifications inside the transaction. Notifications will be handled by a separate `onDocumentCreated` trigger in Plan 04.

    **2. Update `firestore.rules`:**
    - Add rules for `deals/{dealId}` collection:
      ```
      match /deals/{dealId} {
        function isDealParticipant() {
          return request.auth.uid == resource.data.buyerId ||
                 request.auth.uid == resource.data.sellerId;
        }
        allow read: if isAuthenticated() && (isDealParticipant() || isAdmin());
        allow create: if false;  // All creates via Cloud Functions
        allow update: if false;  // All updates via Cloud Functions
        allow delete: if false;

        match /offers/{offerId} {
          allow read: if isAuthenticated() && isDealParticipant();
          allow write: if false;  // All writes via Cloud Functions
        }
      }
      ```
    - Place BEFORE the catch-all deny rule (if one exists), after existing collection rules
    - `isDealParticipant()` is local to the deals match block (not a top-level function since it reads resource.data which is context-dependent)

    **3. Update `firestore.indexes.json`:**

    - Add composite index for `deals` collection: `buyerId` ASC + `updatedAt` DESC, queryScope `COLLECTION` (for My Deals query)
    - Add composite index for `deals` collection: `sellerId` ASC + `updatedAt` DESC, queryScope `COLLECTION` (for My Deals query)
