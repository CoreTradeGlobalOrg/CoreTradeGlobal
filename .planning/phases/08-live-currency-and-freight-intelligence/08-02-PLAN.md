**Props:** `{ price, currency, quantity, estimatedTotal }` — all from the current offer/snapshot. The component is self-contained: it calls useLiveCurrency internally and manages its own target currency state.

**Layout and behavior:**
- Section wrapper: `bg-[#1A283B] border border-[#2A3B52] rounded-xl p-4` (matches existing DealSidebar sections)
- Header: `text-xs font-semibold text-[#8899AA] uppercase tracking-wide mb-3` reading "Currency Conversion"
- Two dropdown selectors for target currencies, each a `<select>` with `bg-[#0F1C2E] border border-[#2A3B52] text-white text-xs rounded px-2 py-1` styling. Options are the 8 TICKER_CURRENCIES (show flag + code). Default: target1=EUR, target2=USD (unless overridden by localStorage).
- On dropdown change: save selected code to localStorage using `TARGET_CURRENCY_KEYS.target1` / `.target2`. Initialize from localStorage on mount inside a useEffect.
- If the deal's base currency matches a target, auto-swap to a different default (e.g., if deal is in EUR, default target1 to USD instead).
- For each target currency, show two rows:
  1. "Unit Price" label + converted unit price (use `convertAmount(price, currency, targetCode, rates)` + format with `Intl.NumberFormat`)
  2. "Est. Total" label + converted estimated total
- Format: gold `#FFD700` for converted amounts, `text-[#8899AA]` for labels
- Exchange rate info line: small text showing "1 {baseCurrency} = {rate} {targetCurrency}" in `text-[#8899AA]`
- Loading state: two skeleton rows matching the animate-pulse pattern
- Stale/error states: if `isStale`, show small amber "(delayed)" badge. If `cacheExpired` or `error`, show "Conversion unavailable" message in muted text.
- If price is null/undefined (no offer yet), show "Awaiting offer data" in muted text.
