**freightConstants.js:**
Create constants for the Freightos integration:
- `FREIGHTOS_BASE_URL = 'https://ship.freightos.com/api/shippingCalculator'`
- `FREIGHTOS_PROXY_URL = '/api/freight/estimate'` (fallback if CORS blocked)
- `FREIGHTOS_LOADTYPES` object: `{ BOXES: 'boxes', PALLETS: 'pallets', CONTAINER_20: 'container20', CONTAINER_40: 'container40', CONTAINER_40HC: 'container40HC' }`
- `TRANSPORT_MODES` array of objects: `{ key, label, icon }` where icon is the lucide-react icon name string ('Ship', 'Plane', 'Truck'). Keys: 'FCL', 'LCL', 'air', 'LTL', 'FTL'. Labels: 'Sea (FCL)', 'Sea (LCL)', 'Air Freight', 'Road (LTL)', 'Road (FTL)'.
- `getChargeableWeight(actualKg, lengthCm, widthCm, heightCm)` — returns `Math.max(actualKg, (L*W*H)/5000)` or just `actualKg` if dimensions not provided
- `suggestLoadType(weightKg)` — heuristic: <100kg => 'boxes', 100-1000kg => 'pallets', >1000kg => 'container20'

**useFreightEstimate.js:**
Create an on-demand fetch hook (NOT polling — only called on user action).

```javascript
export function useFreightEstimate()
// Returns: { estimate, loading, error, rateLimited, fetchEstimate, reset }
```

`fetchEstimate({ origin, destination, weight, loadtype, width, height, length })`:
- First attempt: direct fetch to FREIGHTOS_BASE_URL with URLSearchParams (`origin`, `destination`, `weight`, `loadtype`, `format=json`, `resultSet=cheapestEachMode`). Append width/height/length if provided.
- CORS handling: If the fetch throws a TypeError (CORS blocked) or response is opaque, automatically retry through FREIGHTOS_PROXY_URL (Next.js API route). Log a console.warn noting the CORS fallback. Set an internal `_usedProxy` flag. Add a code comment: `// INTEL-04 tradeoff: proxy fallback means requests count against server IP rate limit (100/hr) instead of per-user IP. Client-side direct calls preferred.`
- If response.status === 429: set `rateLimited = true`, throw a custom error message "Estimate limit reached. Try again in a few minutes."
- Parse response defensively with optional chaining (Pitfall 5). Extract quotes array — for each quote: `{ mode, minPrice, maxPrice, currency, transitDays }`. The response structure may have `response.Quotes[]` with `totalPrice.amount` and `estimatedTotalPrice.amount` (use them as min/max). If only one price field exists, use it for both min and max.
- If response structure is unexpected (no Quotes array or empty), set error to "No freight estimates available for this route."
- `reset()` clears estimate, error, rateLimited states.

**CORS Proxy Fallback — /api/freight/estimate:**

- Accepts GET with same query params (origin, destination, weight, loadtype, format, resultSet, width, height, length)

- Returns the JSON response as-is
- Add a comment: `// INTEL-04 deviation: This proxy exists only as CORS fallback. Ideally all Freightos calls go directly from browser. When proxied, all requests share the server's IP for rate limiting (100 req/hr), reducing per-user budget.`
