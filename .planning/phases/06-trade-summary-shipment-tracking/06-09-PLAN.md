<objective>
Fix the BLOCKER Firestore permission errors preventing member users from viewing Trade Summary and providers from using Active Shipments tab.

Purpose: Tests 9, 11, and 15 all share the same root cause — shipmentTracking subcollection reads fail because (1) the Firestore rules check resource.data fields that queries cannot satisfy, and (2) useActiveShipments queries shipmentTracking without a where clause that matches any rule. The fix: add a `readers` array to each shipmentTracking document (written by Cloud Functions), simplify Firestore rules to isAuthenticated() for these subcollections, and add `where('readers', 'array-contains', uid)` to client queries for the shipmentTracking index.

Output: Working Trade Summary for members, working Active Shipments for providers, working print without localhost URL.
</objective>
