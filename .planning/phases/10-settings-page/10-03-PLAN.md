**src/app/api/subscription-status/route.js:**
- GET endpoint accepting `?email=` query param
- Lowercase and trim the email
- Hash email with SHA-256 to get doc ID (same pattern as Phase 9 unsubscribe CF)
- Use `getAdminFirestore()` from `@/lib/firebase-admin` to check if doc exists in `unsubscribes` collection
- Return `{ subscribed: !snap.exists }` (true = not unsubscribed, false = unsubscribed)
- If no email param, return `{ subscribed: true }` (default)

**src/presentation/hooks/settings/useNotificationPreferences.js:**
- Export `useNotificationPreferences(uid)` hook
- Import `doc`, `onSnapshot`, `updateDoc` from firebase/firestore, `db` from firebase.config
- On mount: subscribe to `users/{uid}` document via onSnapshot
- Extract `preferences` field from doc data, with defaults: all categories enabled for both email and push
- Default shape if preferences is undefined: `{ deals: {email:true, push:true}, messages: {email:true, push:true}, legal: {email:true, push:true}, providers: {email:true, push:true}, system: {email:true, push:true} }`
- `updatePreference(category, channel, value)` async: calls `updateDoc(doc(db, 'users', uid), { ['preferences.' + category + '.' + channel]: value })` with toast on success
- Return `{ preferences, updatePreference, loading }`

**src/presentation/hooks/settings/useEmailSubscriptions.js:**
- Export `useEmailSubscriptions(email)` hook
- State: `subscribed` (boolean, default true), `loading`, `updating`
- On mount: fetch `GET /api/subscription-status?email=${encodeURIComponent(email)}` to get initial state
- `toggleSubscription()` async:
  - If currently subscribed: POST `/api/unsubscribe` with body `{ email }` -> set subscribed=false
  - If currently unsubscribed: POST `/api/resubscribe` with body `{ email }` -> set subscribed=true
  - Toast on success, catch errors
- Return `{ subscribed, toggleSubscription, loading, updating }`
