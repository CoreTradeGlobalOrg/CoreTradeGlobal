  <action>
1. Create `OnboardingTour.jsx` — a React Portal-based overlay tour component:
   - Props: `{ user, onComplete }` — user object with role and uid, callback when tour finishes
   - Renders via `createPortal` to `document.body` to avoid z-index stacking issues
   - Tour steps defined as two arrays (trade vs provider):
     - **Trade companies (role=member):** 1) "Dashboard Overview" — targets the main content area, explains deal tracking and activity, 2) "Browse Products" — targets the Products nav link, explains marketplace, 3) "Submit an RFQ" — targets the Requests nav link, explains quote requests, 4) "Start a Deal" — targets the deals section or Deals nav link, explains negotiation flow, 5) "Messages" — targets the Messages nav link or FAB widget, explains communication
     - **Providers (role=logistics_provider or insurance_provider):** 1) "Dashboard Overview" — targets main content area, 2) "Quote Requests" — targets the provider dashboard requests tab, 3) "Submit a Quote" — explains how to respond to requests, 4) "Active Shipments/Policies" — targets active tab, 5) "Messages" — targets the Messages nav or FAB
   - Each step: `{ targetSelector: string, title: string, description: string, placement: 'bottom'|'top'|'left'|'right' }`
   - State: `currentStep` (index), `targetRect` (from getBoundingClientRect)
   - On step change: find target element via `document.querySelector(step.targetSelector)`. If element not found, skip to next step. Use `requestAnimationFrame` to defer rect calculation to after paint.
   - Overlay: full-screen fixed div with `z-[9990]`. Use `box-shadow: 0 0 0 9999px rgba(0,0,0,0.75)` on a spotlight div positioned at `targetRect` to create the cutout effect. The spotlight div has `pointer-events: none`.
   - Step panel: positioned near the target element based on `placement`. Styled with glass-card pattern: `bg-[#1A283B]/95 backdrop-blur-md border border-[#2A3B52] rounded-xl p-5 shadow-2xl`. Max-width 320px.
   - Panel content: step title (text-white font-bold), description (text-gray-400 text-sm), step counter ("Step X of 5"), Next/Skip buttons. Next button: gold gradient. Skip button: text-gray-400 underline.
   - On "Skip tour" or after final step "Finish": call `updateDoc(doc(db, 'users', user.uid), { onboardingTourCompleted: true, onboardingTourCompletedAt: new Date() })`, then call `onComplete()`.

2. Create `OnboardingTour.css`:
   - `.onboarding-overlay` — `position: fixed; inset: 0; z-index: 9990; pointer-events: none;`
   - `.onboarding-spotlight` — absolute positioned div with the box-shadow cutout, `border-radius: 8px; transition: all 0.3s ease;`
   - `.onboarding-panel` — `pointer-events: auto;` (re-enables clicks on the panel itself)
   - Smooth transition for spotlight movement between steps

3. Integration point: The tour will be rendered in the dashboard/homepage. In `src/app/(main)/page.js` (or the main layout), conditionally render `<OnboardingTour>` when `user && !user.onboardingTourCompleted`. Use `next/dynamic` with `ssr: false` for the tour component since it uses DOM APIs (getBoundingClientRect, createPortal). Check the user doc field — if `onboardingTourCompleted` is falsy/missing, show the tour.
