Perform a systematic WCAG 2.1 AA accessibility audit and apply fixes. This is a sweep task — the executor should:

1. Run Lighthouse accessibility audit on these key pages (via `npx next dev` in a separate terminal):
   - Homepage (`/`)
   - Register page (`/register`)
   - Login page (`/login`)
   - Products/marketplace page (`/products`)
   - Profile page (`/profile/[userId]`)
   - Deals page (`/deals`)
   - Settings page (`/settings`)

2. Fix categories of issues found (common patterns in this codebase based on prior phases):

   **Images and media:**
   - Add `alt` attributes to all `<img>` tags and `next/image` components that are missing them
   - Decorative images get `alt=""`, meaningful images get descriptive alt text
   - Company logos: `alt="{companyName} logo"`
   - Product images: `alt="{productName}"`

   **Interactive elements:**
   - Add `aria-label` to icon-only buttons (star buttons, close buttons, menu toggles)
   - Ensure all buttons have accessible names (not just icons)
   - Add `role="button"` to clickable divs that should be buttons (or convert to `<button>`)

   **Forms:**
   - Ensure all form inputs have associated `<label>` elements or `aria-label`
   - Add `aria-describedby` linking inputs to their error messages
   - Ensure error messages have `role="alert"` or `aria-live="polite"`

   **Keyboard navigation:**
   - Add `tabIndex="0"` to custom interactive elements that should be keyboard-focusable
   - Ensure focus indicators are visible: add `focus-visible:ring-2 focus-visible:ring-[#FFD700]` to interactive elements missing focus styles
   - Ensure modal dialogs trap focus (check existing Modal component)

   **Color contrast:**
   - Check `text-gray-400` on dark backgrounds — may need to upgrade to `text-gray-300` for 4.5:1 ratio
   - Check gold text `text-[#FFD700]` on dark backgrounds — usually passes, verify
   - Fix any instances where text-on-text color contrast is below 4.5:1

   **Semantic HTML:**
   - Ensure pages have exactly one `<h1>`
   - Check heading hierarchy (no skipping from h1 to h3)
   - Add `<main>` landmark if missing
   - Add `aria-current="page"` to active nav links

   **Screen reader:**
   - Add `sr-only` text for icon-only actions
   - Add `aria-live="polite"` to loading states and dynamic content areas
   - Add `aria-expanded` to expandable sections (accordions, dropdowns)

3. Apply fixes directly in the affected files. Prioritize: critical violations first, then serious, then moderate. Do NOT change visual design — only add accessibility attributes and minor CSS for focus indicators.

4. Keep a running list of files modified for the SUMMARY.
