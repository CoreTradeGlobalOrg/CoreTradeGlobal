<task type="auto">
  <name>Task 1: Create sendAnnouncement and processScheduledAnnouncements Cloud Functions + Firestore rules</name>
  <files>functions/index.js, firestore.rules</files>
  <action>
1. **`sendAnnouncement` onCall CF**: Admin-only (verify caller is admin via custom claims). Accepts `{ title, body, audience, channels, scheduledFor }`.
   - If `scheduledFor` is provided and is in the future: write to `announcements` collection with status: 'pending', return `{ status: 'scheduled' }`
   - If `scheduledFor` is null or in the past: execute immediately:
     a. Query users by audience: if 'all' -> all users; if specific role -> query where role == audience
     b. For each targeted user:
        - If `channels.inApp`: create notification doc in `users/{uid}/notifications` with type: 'announcement', title, body, link: '/notifications'
        - If `channels.push` and user has fcmToken and `preferences?.system?.push !== false`: send FCM push via `messaging.send`
        - If `channels.email` and `preferences?.system?.email !== false`: send email via `sendDealEmail` using `buildBrandedEmailHtml` (from Plan 02) with the announcement body content
     c. Write to `announcements` collection with status: 'sent', sentAt: Timestamp.now(), recipientCount
     d. Return `{ status: 'sent', recipientCount }`
   - All sends are non-blocking (try/catch per user). Invalid FCM token cleanup on send errors.

2. **`processScheduledAnnouncements` onSchedule CF**: Runs every 5 minutes (matches existing `updateFairStatuses` pattern).
   - Query `announcements` where `status === 'pending'` and `scheduledFor <= Timestamp.now()`
   - For each: execute the same send logic as immediate announcements (extract into a shared `deliverAnnouncement(announcementData)` helper to avoid duplication)
   - Update announcement doc: `status: 'sent'`, `sentAt: Timestamp.now()`, `recipientCount`
   - On error: set `status: 'failed'`, log error

3. **Firestore rules for `announcements` collection**:
   - Read: admin only (`request.auth != null && request.auth.token.role == 'admin'`)
   - Write (create/update): admin only
   - Delete: deny (announcements are historical records)
  </action>
