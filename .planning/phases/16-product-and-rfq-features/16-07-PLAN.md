  <files>src/presentation/components/features/admin/ProductsRequestsManager/BulkProductUpload.jsx</files>
  <action>
The root cause of GAP-7 is that BulkProductUpload.jsx validates CSV category values against COMPANY_CATEGORIES (company business types like 'automotive', 'electronics') instead of the actual product categories stored in Firestore's `categories` collection.

1. Remove the COMPANY_CATEGORIES import and the static VALID_CATEGORY_VALUES / CATEGORY_LABEL_TO_VALUE derived from it.

2. Add a prop `categories` to the BulkProductUpload component (the parent ProductsRequestsManager already has access to categories via useCategories or can pass them down).

3. Build the category validation maps dynamically from the `categories` prop:
   ```jsx
   // Inside the component, derive from categories prop
   const categoryMaps = useMemo(() => {
     const validValues = new Set();
     const labelToValue = {};
     (categories || []).forEach((c) => {
       validValues.add(c.value || c.id);
       const label = c.name || c.label || '';
       if (label) {
         labelToValue[label.toLowerCase()] = c.value || c.id;
       }
       if (c.value) {
         labelToValue[c.value.toLowerCase()] = c.value;
       }
       if (c.id) {
         labelToValue[c.id.toLowerCase()] = c.id;
       }
     });
     return { validValues, labelToValue };
   }, [categories]);
   ```

4. Update the `resolveCategory` function to use `categoryMaps.labelToValue` instead of the static CATEGORY_LABEL_TO_VALUE.

5. Update `validateRow` to use `categoryMaps.validValues` instead of VALID_CATEGORY_VALUES. The category validation line (around line 58-60) should check `categoryMaps.validValues.has(resolvedCategory)` or rely on `resolveCategory` returning null for invalid values.

6. Update the preview table category display (around line 388) to look up category labels from the `categories` prop instead of COMPANY_CATEGORIES.

7. In `ProductsRequestsManager.jsx`, pass the categories to BulkProductUpload:
   - The component likely already fetches categories or can use `useCategories()` hook
   - Add `categories={categories}` prop to `<BulkProductUpload categories={categories} />`
   - If categories are not available in the parent, add `import { useCategories } from '@/presentation/hooks/category/useCategories'` and call it

   b. Ensure `stockQuantity` field is set (not just `quantity`). The CF currently stores `quantity: row.quantity` but the ProductCard reads `product.stockQuantity`. Change to store BOTH:
      ```js
      quantity: row.quantity,
      stockQuantity: row.quantity,
      ```

   c. Check the `imageUrls` field handling. The CF reads `row.imageUrls` (line 5211) but the client sends raw CSV data. If the client sends raw CSV row objects (with keys like "Product Name", "Category", "Price"), but the CF expects `{ name, categoryId, price, currency, quantity, unit, description, imageUrls }`, this IS the internal error — the CF tries to read `row.name` which is undefined.

      ```js
        description: row['Description']?.trim() || '',

      }));
      ```
