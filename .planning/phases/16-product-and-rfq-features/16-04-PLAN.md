<task type="auto">
  <name>Task 2: bulkUploadProducts Cloud Function with image downloading</name>
  <files>
    functions/index.js

  <action>
1. Add `bulkUploadProducts` onCall Cloud Function to `functions/index.js`:
   - **Auth check**: Verify caller has admin custom claim (`request.auth.token.role === 'admin'`). Reject non-admins.
   - **Input**: `{ userId: string, rows: Array<{ name, categoryId, price, currency, quantity, unit, description, imageUrls }> }`.
   - **For each row**:
     a. Create a product document in `products` collection with the provided fields + `userId` (the selected member), `status: 'active'`, `createdAt: FieldValue.serverTimestamp()`, `updatedAt: FieldValue.serverTimestamp()`.
     b. For each image URL in `imageUrls` (split by comma, trim whitespace):
        - Use `node-fetch` (or the built-in `fetch` if Node 18+) to download the image with a 10-second timeout.
        - If fetch succeeds and returns a 2xx status, upload the buffer to Firebase Storage at path `{userId}/products/{productId}/image_{index}.{ext}`.
        - Make the file public or generate a download URL using `getDownloadURL` pattern.
        - If fetch fails (timeout, 404, CORS), skip that image and log the error. Do not fail the entire row.
     c. Update the product document's `images` array with the successfully downloaded image URLs.
   - **Return**: `{ created: number, skipped: number, errors: Array<{ row: number, reason: string }> }`.
   - **Rate limiting**: Process rows sequentially (not all in parallel) to avoid overwhelming external image servers. Use a small concurrency limit (e.g., 3 rows at a time).

2. Check if `node-fetch` is already in `functions/package.json`. If not, add it. If the functions runtime is Node 18+, use built-in `fetch` instead.

3. Ensure the Cloud Function has appropriate timeout configured (300s for large uploads) via `runWith({ timeoutSeconds: 300 })` or the v2 equivalent.
