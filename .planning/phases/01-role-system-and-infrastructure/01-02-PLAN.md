**1. Create `OnboardingWizard.jsx`:**

Multi-step wizard component with 4 steps (per user decision: set password, confirm details, add photo, configure preferences):

**Step 1 — Verify Identity & Set Password:**
- Check if the current URL is a valid sign-in link using `isSignInWithEmailLink(auth, window.location.href)` from Firebase client SDK
- If `emailForSignIn` is in localStorage (same device), use it automatically
- If not (cross-device scenario per Pitfall 5 in RESEARCH.md), show an email input field and ask user to enter their email
- Call `signInWithEmailLink(auth, email, window.location.href)` to complete sign-in
- After successful sign-in, show a password field + confirm password field
- Call `updatePassword(auth.currentUser, password)` to set their password
- After password set, force token refresh: `auth.currentUser.getIdToken(true)`
- Then update session cookie by calling `/api/auth/session` POST with the fresh token
- This ensures custom claims are in the session from this point forward
- Use react-hook-form + zod for password validation (min 8 chars, etc.)

**Step 2 — Confirm Details:**
- Fetch the user's Firestore `users/{uid}` doc (created by inviteUser CF)
- Display read-only: name, company, role (with role badge from roles.js)
- Show email (read-only)
- Allow user to edit displayName if they want (optional)
- This step is confirmation — user sees what role they've been assigned

**Step 3 — Profile Photo:**
- File upload input for profile photo
- Follow existing `uploadCompanyLogo` pattern from the codebase (check FirebaseStorageDataSource or similar)
- Upload to Firebase Storage at path `users/{uid}/profile.{ext}`
- Update Firestore `users/{uid}` doc with `photoURL` field
- Also update Firebase Auth profile: `updateProfile(auth.currentUser, { photoURL })`
- Show preview of uploaded photo
- Make this step skippable ("Skip for now" button)

**Step 4 — Preferences & Complete:**
- Simple preferences form:
  - Email notifications toggle (default on)
  - Any role-specific preferences (can be minimal for v1)
- Save preferences to Firestore `users/{uid}` doc
- On completion:
  - Update invite doc status to 'accepted' and set acceptedAt timestamp
  - Update user doc: `inviteStatus: 'accepted'`, `onboardingComplete: true`
  - Redirect to role-appropriate page:
    - `logistics_provider` or `insurance_provider` -> `/provider`
    - `lawyer` -> `/lawyer`
    - Default -> `/`

**Wizard UI:**
- Use framer-motion for step transitions (AnimatePresence, slide left/right)
- Step indicator at top showing progress (Step 1 of 4, Step 2 of 4, etc.)
- Use lucide-react icons for step indicators
- Back button (except on step 1), Next/Continue button, Skip where applicable
- Tailwind styling consistent with existing auth pages (check login/register for color palette)
