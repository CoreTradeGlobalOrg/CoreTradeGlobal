**2. `saveDraftApprovals` â€” onCall Cloud Function:**

Accepts `{ dealId, approvedClauses }` from `request.data`. Auth required (`request.auth?.uid`).

Guards:
- Auth guard: `if (!uid) throw HttpsError('unauthenticated')`
- Read contract doc: `if (!contractDoc.exists) throw HttpsError('not-found', 'Contract not found.')`
- Check participation using denormalized fields on contract: `if (uid !== contractDoc.data().dealBuyerId && uid !== contractDoc.data().dealSellerId) throw HttpsError('permission-denied')`
- If party has already submitted (`hasSubmitted === true`), throw HttpsError('failed-precondition', 'Cannot modify after submission.')

Determine `isBuyer` from `contractDoc.data().dealBuyerId === uid` (uses the denormalized field on the contract document â€” no need to read the deal doc separately). Compute `approvalKey` = `buyerApproval.approvedClauses` or `sellerApproval.approvedClauses`.

Update contract doc with: `{ [approvalKey]: approvedClauses }` (replaces full array â€” NOT arrayUnion, since we're saving the complete draft state).

This is NOT inside a transaction (draft saves are best-effort per research). Return `{ success: true }`.

**3. `submitContractApproval` â€” onCall Cloud Function:**

Accepts `{ dealId }` from `request.data`. Auth required.

MUST use `db.runTransaction` (atomic â€” prevents the race condition where both parties submit simultaneously and both read `otherHasSubmitted = false`).

Inside the transaction:
1. Read both `dealRef` and `contractRef` (in parallel with `Promise.all([t.get(dealRef), t.get(contractRef)])`)
2. Guards:
   - Deal exists, contract exists
   - `uid === deal.buyerId || uid === deal.sellerId` (participation)
   - `deal.status === 'accepted'` (not already contract_approved)
   - Party's `hasSubmitted === false` (cannot re-submit)
   - Party's `approvedClauses.length === contract.clauses.length` (must approve all clauses)
3. Determine `isBuyer`, `approvalKey`, `otherApprovalKey`
4. Build update data:
   - Set `${approvalKey}.hasSubmitted = true`
   - Set `${approvalKey}.submittedAt = Timestamp.now()`
5. Check if other party has already submitted (`contract[otherApprovalKey].hasSubmitted`):
   - If yes: set `contract.status = 'both_approved'` AND update deal doc `status = 'contract_approved'`, `updatedAt = Timestamp.now()`
   - If no: set `contract.status = isBuyer ? 'buyer_approved' : 'seller_approved'`
6. Apply updates via `t.update(contractRef, updateData)` and optionally `t.update(dealRef, ...)`

Return `{ success: true }`.

**After the transaction** (OUTSIDE, not inside â€” per established pattern), send notifications:
- Call `sendDealNotifications(dealId, 'contract_approved_by_party', uid, deal)` when one party submits
- Call `sendDealNotifications(dealId, 'contract_both_approved', uid, deal)` when both parties have submitted

**4. Extend `sendDealNotifications` event type handling:**
